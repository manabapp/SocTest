Chapter 5. Interface Configuration
SOCTEST - USER'S GUIDE

本章では、インターフェースの表示機能について説明します。

Interface Configuration 画面では、端末のネットワークインターフェースのアドレス表示および状態確認を行います。
また、表示内容を最新の状態に更新します。

アドレスおよび状態の確認

iOS 端末で IPv4 通信が可能なネットワークインターフェースの各種情報を表示します。
これにより、サーバーソケットに割り当てるアドレスやブロードキャスト通信で指定するアドレスを確認できます。
  • IPv4 アドレス
  • インターフェース識別子
  • インターフェース名
  • ネットマスク
  • ブロードキャストアドレス
  • MAC アドレス（ダミーアドレス 02:00:00:00:00:00 が表示されます）
  • 状態フラグ（フラグ名がカンマ区切りで表示されます）
  • FQDN（IPv4 アドレスから DNS 逆引きができた場合のみ表示されます）
利用可能なネットワークインターフェースについて以下に説明します。

ネットワーク
インターフェース		説明
Cellular通信キャリアの移動体通信網（Mobile carrier network）に接続して IPv4 通信を行います。
端末の下記設定が有効になっている必要があります。

  • [設定] > [モバイル通信] > [SocTest]、または
  • [設定] > [SocTest] > [モバイルデータ通信]
Wi-fiWi-Fi ネットワークに接続して IPv4 通信を行います。
Personal Hotspotインターネット共有（以降、Personal Hotspot）機能とは、iOS 端末をモバイルルーターとして動作させ、他の端末を Cellular ネットワークに接続（中継）する機能です。
この機能を利用して、ローカルネットワーク（172.20.10.0/28）上で他端末間と IPv4 通信することが可能です。
インターフェースの IP アドレスは、172.20.10.1 固定です。

端末の下記設定が有効になっている必要があります。

  • [設定] > [プライパシー] > [ローカルネットワーク] > [SocTest]、または
  • [設定] > [SocTest] > [ローカルネットワーク]

Loopback127.0.0.1（ホスト名: localhost）を使用して IPv4 ループバック通信を行います。


表示内容の更新

更新する をタップすると端末の設定を再読み込みします。
現在地の移動によるモバイル通信基地局の切り替わりや Wi-Fi に接続した場合などに、新しいアドレスを確認できます。
